🔥 三大答題鐵則（老師原話 → 拿 first class 70+ 的關鍵）

1. 公式 + LHS（左邊一定要寫）

"If you only write the formula floating without the LHS, the math is not formally coherent — you cannot get full marks."

寫公式時務必加上左邊，例如：

  • ✅ sim(w₁, w₂) = (w₁·w₂) / (‖w₁‖·‖w₂‖)
  • ✅ P(w_k | w_{k-1}) = count(w_{k-1}, w_k) / count(w_{k-1})
  • ❌ 只寫右邊一坨 — 不算 formally coherent，拿不到滿分。
2. 用 formal terminology（這幾個詞會被閱卷者圈起來）

看到題目就有反射：

  • distributional hypothesis（不要寫 "words have similar meanings"）
  • continuation probability（不要寫 "how often a word appears"）
  • Markov assumption（不要寫 "only depends on previous"）
  • conditional independence（不要寫 "naive"）
  • generative / discriminative（必須明確說哪一派）
  • maximum likelihood（不要寫 "count and divide"）
  • cross-entropy（不要只寫 "log loss"）

這些字寫出來就是 70+，不寫就是 60。

📊 Q2 Language Modelling · 12 cards
Q2Language Model點擊翻面 ↻
A probability distribution over token sequences; assigns $P(w_1,...,w_n)$ to any sequence over vocabulary $V$.句子的人話偵測器
Q2N-gram點擊翻面 ↻
A contiguous sequence of N tokens; basic unit of count-based LMs (unigram, bigram, trigram).N 個字一組
Q2Markov Assumption點擊翻面 ↻
The next token depends only on a bounded k-window of preceding tokens: $P(w_i|history) \approx P(w_i|w_{i-k},...,w_{i-1})$.失憶症：只看前 k 個
Q2MLE點擊翻面 ↻
Observed relative frequency; for bigrams: $P(w_k|w_{k-1}) = c(w_{k-1},w_k) / c(w_{k-1})$.數數法
Q2Smoothing點擊翻面 ↻
Redistributes probability mass from observed n-grams to unobserved ones, preventing zero probabilities.給沒看過的也分一點機率
Q2Add-1 (Laplace)點擊翻面 ↻
Add pseudo-count of 1 to every n-gram. Over-smooths when |V| is large.P = (c+1) / (c(·)+|V|)每個加 1，太大方
Q2Add-k點擊翻面 ↻
Generalisation of Add-1 with tunable k ∈ (0,1].P = (c+k) / (c(·)+k·|V|)Add-1 的小氣版
Q2Good-Turing點擊翻面 ↻
Re-estimates count using frequency of frequencies: $r^* = (r+1) N_{r+1}/N_r$. Unseen prob ≈ $N_1/N$.用 singletons 估 unseen
Q2Interpolated Kneser-Ney點擊翻面 ↻
Absolute discount + back-off to continuation probability. 3 parts: d, λ, P_cont.P = max(c-d,0)/c + λ·P_contFrancisco 只跟 San → P_cont 低
Q2Continuation Probability點擊翻面 ↻
Fraction of distinct bigram types where $w$ appears as the second word.|{w': c(w',w)>0}| / |distinct bigrams|字的人緣：能配多少前面字
Q2Perplexity點擊翻面 ↻
Inverse geometric mean of per-token probability. Lower = better.PP(W) = P(w₁...w_N)^(-1/N) = 2^H(W)模型多迷茫
Q2Intrinsic vs Extrinsic點擊翻面 ↻
Intrinsic: property of model (perplexity). Extrinsic: downstream task performance (BLEU, WER).內測 vs 外賽
🎯 Q3 Embeddings + Classification · 14 cards
Q3One-hot Vector點擊翻面 ↻
Sparse vector of length |V| with a single 1. Carries no semantic info (all pairs orthogonal).投票紙只勾一格
Q3Word Embedding點擊翻面 ↻
Dense, low-dimensional, distributed vector; semantically similar words have geometrically close vectors.每個字的 GPS 座標

Q3CBOW點擊翻面 ↻
Word2Vec variant: predicts the centre word from averaged surrounding context. Faster.填空題
Q3Skip-gram點擊翻面 ↻
Word2Vec variant: predicts each context word from the centre word. Better on rare words.反向填空題
Q3Negative Sampling點擊翻面 ↻
Replaces full softmax with binary classification: sample $k$ negative pairs from unigram dist. O(k) not O(|V|).1 真 + 5 假對比
Q3GloVe點擊翻面 ↻

Q3TF-IDF點擊翻面 ↻

Model fits training noise; train error low, validation error high. Fix: regularisation, dropout, early stopping.背題目背太熟

Stochastic process $(Q, A, \pi)$ — states + transition matrix + initial distribution. Markov property: next depends only on current.失憶跳棋

4-tuple $(Q, A, B, \pi)$: Markov chain on hidden states + emission matrix B generating observations.雙層蛋糕：天氣→冰淇淋

2. Tokenisation + Maximum Matching + Sentence Segmentation'22'23'24'25
把連續文字切成電腦能吃的小塊（token）。英文靠空白切；中文要靠字典（Max Matching）；切句要判斷句點是不是真句尾。

Tokenisation 定義

Tokenisation is the process of splitting text into atomic units called tokens, where a token may be a subword, character, word, sentence, or other granularity defined by the model's vocabulary.

主要挑戰（必背舉例）

  • Apostrophes / contractions: don't → do + n't?
  • Hyphenated forms: state-of-the-art 一個 token 還是四個？
  • Multi-word proper names: New York 切開就壞了
  • Languages without spaces: 中文、日文、泰文 — 用 Maximum Matching 或 neural segmenter
  • URLs / emails / hashtags / emojis: 簡單 whitespace split 全毀

Maximum Matching（必背演算法 — '24 連兩年考）

Maximum Matching is a greedy left-to-right algorithm that, at each position, emits the longest substring matching a dictionary entry; used for languages without explicit word delimiters (e.g. Mandarin).

步驟：

  1. 從位置 0 開始
  2. 找出從該位置開始字典中最長能 match 的字
  3. 輸出該字，pointer 跳到該字之後
  4. 重複到結尾；無 match 時退回單字元

Sentence Segmentation

Sentence segmentation splits running text into sentences. Treated as a binary classification problem: for each ./?/!, decide whether it ends a sentence.

難點：句點不一定是句尾

  • abbreviations（Dr., U.S.A., Jr.）
  • decimals（3.14）
  • URLs（www.example.com）
  • name initials（J.K. Rowling）

features：前一 token 是否為已知縮寫；後一 token 是否大寫開頭；前後有無空白；前一 token 長度。

📜 過去考題：'24 Q1(d) — Maximum Matching 應用

Q：Apply Maximum Matching to "Thetabledownhere" assuming dict = {The, the, table, down, here, ...}

A：

  1. pos 0：最長 match = The → 輸出，pos → 3
  2. pos 3：最長 match = table → 輸出，pos → 8
  3. pos 8：最長 match = down → 輸出，pos → 12
  4. pos 12：最長 match = here → 輸出

輸出： The / table / down / here

注意 issues：local ambiguity（貪婪可能切錯）、OOV（字典外退單字元）、case sensitivity、no right context（不 look ahead）。

3. Lemmatisation vs Stemming（Porter）'20'21'22'24'25
兩個都是把字「還原」。Lemmatisation 嚴格用字典 — 輸出一定是真實字（are → be）；Stemming 用規則砍後綴 — 輸出可不是真字（computational → comput）。

滿分定義

Lemmatisation is the reduction of an inflected word to its lemma (the dictionary headword), requiring a lexicon and morphological analysis (often POS). The output is always a real word.
Stemming is the rule-based truncation of a word's suffixes to produce a stem, which need not be a real word. Different stemmers (Porter, Snowball, Lancaster) yield different stems.

5. WordNet + Lesk Algorithm（WSD）'21'25
WordNet 是英文字典 + 朋友圈關係圖。Lesk 是「重疊比對」的偵探算法 — 哪個 sense 的字典定義跟上下文重疊最多字，就選那個。

WordNet 定義

WordNet is a large lexical database of English organised into synsets (sets of cognitive synonyms), linked by semantic relations such as hypernymy, meronymy, antonymy.

Lesk Algorithm 定義

Lesk algorithm performs word sense disambiguation by selecting the sense whose gloss (dictionary definition) has the largest word overlap with the context (or with another word's gloss).

步驟（很重要 — 兩年都考）

  1. 找出目標 ambiguous word 的所有 senses 跟對應 glosses
  2. 取出當前 context（或另一個 target word 的 gloss）的內容詞
  3. 對每個 sense 計算 gloss ∩ context 的字數（去除 stopwords）
  4. 選 overlap 最大的 sense

6. Language Model + N-gram + Markov + MLE'20'22'23'24
LM 是「句子機率打分器」。給任何 token 序列吐一個機率。N-gram 是把序列切成 N 個一組的 token 串；Markov assumption 把長依賴近似成只看前 k 個；MLE 用觀察到的相對頻率估機率。

LM 定義（滿分版）

Language Model (LM): a probability distribution over sequences of tokens that assigns $P(w_1, ..., w_n)$ to any token sequence drawn from a vocabulary $V$. Not merely a next-word predictor.
"I would love to think that no one is going to get out of this module thinking that language models started with ChatGPT. Statistical language models have been around for decades."

3 大 LM 應用（要會講）

  1. Speech recognition (ASR)：re-rank candidate transcripts by LM probability
  2. Machine translation (MT)：re-rank candidate translations
  3. Spell correction / autocomplete：choose corrections / next-word that maximise likelihood

N-gram 與 Markov assumption

NamenMarkov order例
Unigram10（無歷史）(I), (love), (NLP)
Bigram21(I, love), (love, NLP)
Trigram32(I, love, NLP)
Markov assumption (k-th order): $P(w_i \mid w_1, ..., w_{i-1}) \approx P(w_i \mid w_{i-k}, ..., w_{i-1})$ — 下一個字只看前 k 個。

MLE — 必背

Bigram MLE: $P_{\text{MLE}}(w_k \mid w_{k-1}) = \dfrac{\text{count}(w_{k-1}, w_k)}{\text{count}(w_{k-1})}$
Sentence probability (bigram): $P(s) = P(w_1 \mid \langle s\rangle) \cdot \prod_{i=2}^{n} P(w_i \mid w_{i-1}) \cdot P(\langle/s\rangle \mid w_n)$

長 n-gram 的問題（'23 考過）

  1. Data sparsity: $|V|^n$ 個可能組合，絕大多數沒見過 → MLE 不可靠或為 0
  2. Storage: 儲存所有 n-gram 不切實際
  3. Diminishing returns: bias-variance tradeoff — 長 n-gram 噪聲大

Log 轉換為什麼用？

N-gram 機率小（$10^{-3}$–$10^{-5}$）；連乘 30 個會 underflow。解：$\log \prod_i P(w_i | w_{i-1}) = \sum_i \log P(w_i | w_{i-1})$。乘法變加法，數值穩定。

7. Smoothing 四方法（Add-1, Add-k, Good-Turing, Interpolated KN）'20'21'22'23'24'25 🔥 每年
Smoothing 是「公平政策」：把一些機率分給沒看過的字組合，避免整句機率變 0。不同方法是分配公平度的不同設計。Interpolated KN 是必背的王者方法。
"There has never been a past exam without at least one exercise about smoothing."

定義

Smoothing: any technique that redistributes probability mass from observed n-grams to unobserved ones, ensuring no n-gram receives zero probability.

① Add-1 (Laplace) — 最簡單也最常考

$P_{\text{Add-1}}(w_k \mid w_{k-1}) = \dfrac{c(w_{k-1}, w_k) + 1}{c(w_{k-1}) + |V|}$

主要問題：$|V|$ 在 NLP 通常 $10^4$–$10^6$，分母被 $|V|$ 主導 → 觀察到的 bigram 機率被大幅稀釋（over-smoothing）。

② Add-k — 廣義版

$P_{\text{Add-}k}(w_k \mid w_{k-1}) = \dfrac{c(w_{k-1}, w_k) + k}{c(w_{k-1}) + k|V|}$, $k \in (0, 1]$

③ Good-Turing — 用稀有事件估 unseen

$r^* = (r+1) \dfrac{N_{r+1}}{N_r}$   where $N_r$ = number of n-grams seen $r$ times.
$P_{\text{GT}}(\text{unseen}) = N_1 / N$

直覺：如果 corpus 裡有很多事件只出現 1 次（$N_1$ 大），那 unseen 事件的真實機率應該不會是 0。

④ Interpolated Kneser-Ney — 滿分必背

背完整公式 + 三要素，全題 6 分穩拿：
Full formula:
$P_{\text{KN}}(w_i \mid w_{i-1}) = \dfrac{\max(c(w_{i-1}, w_i) - d, 0)}{c(w_{i-1})} + \lambda(w_{i-1}) \cdot P_{\text{cont}}(w_i)$
要素定義
① Discount $d$固定折扣常數（典型 0.75）— 從每個觀察到的 count 砍 $d$，騰出機率給 unseen
② Interpolation weight $\lambda(w_{i-1})$$\lambda(w_{i-1}) = \dfrac{d}{c(w_{i-1})} \cdot |\{w: c(w_{i-1}, w) > 0\}|$ — 騰出的機率質量總和
③ Continuation prob $P_{\text{cont}}(w)$$P_{\text{cont}}(w) = \dfrac{|\{w': c(w', w) > 0\}|}{|\{(w', w''): c(w', w'') > 0\}|}$ — w 當第二個字的 distinct bigram 比例
🌟 Francisco 直覺 — 必背

Francisco 出現頻率高，但 幾乎只跟在 San 後面。

普通 back-off 用 unigram 頻率會錯誤地給 Francisco 在任何 context 下高機率。

KN 的 continuation probability 看「有多少不同的字 precede w」 — Francisco 拿到很低的分數，正確結果。

8. Perplexity + Intrinsic/Extrinsic Evaluation'20'22'24
Perplexity 是「模型有多迷茫」— 對 test set 的 inverse geometric mean。低 = 好。等於 cross-entropy 取指數，可理解為 effective branching factor。

必背公式

Cross-entropy: $H(W) = -\dfrac{1}{N} \log_2 P(w_1, ..., w_N)$
Perplexity: $PP(W) = P(w_1, ..., w_N)^{-1/N} = 2^{H(W)}$

關鍵特性（必背）

  1. Lower = better model（more confident，less perplex）
  2. 單一值無意義 — perplexity is a comparison metric；同 test set 跨 model 比較才有意義
  3. 等於 cross-entropy 的指數 — effective branching factor per token
  4. 缺點：只能比較相同 vocabulary 的 models；OOV 處理敏感

📜 過去考題：'20 Q2(d-iii) — Perplexity 公式 + 解釋 [6]

滿分答：

Formula: $PP(W) = P(w_1, ..., w_N)^{-1/N} = \sqrt[N]{1/\prod_{i=1}^N P(w_i|w_{i-1})}$ where N = total bigrams in test text.

How it evaluates:

  • Lower perplexity = better model — higher sentence probability means LM is more confident (less perplexed).
  • Perplexity is the exponential of cross-entropy — interpreted as effective branching factor per token.
  • Comparison-only: a single perplexity is meaningless; values comparable only on same test set with same vocabulary.

Negative Sampling (SGNS)

Negative Sampling replaces the full softmax with binary classification. For each true (centre, context) pair, sample $k$ negative pairs from the unigram distribution; train to discriminate true from negative.

Loss: $\log \sigma(v_c \cdot v_w) + \sum_{i=1}^{k} \log \sigma(-v_{w_i} \cdot v_c)$

為什麼用：full softmax 對 $|V| \approx 10^5$ 太慢，negative sampling 把 $O(|V|)$ 降到 $O(k)$（$k \approx 5\text{-}20$），品質類似。

  2. Quality: produces high-quality embeddings comparable to full softmax in practice.
  3. Implicit matrix factorisation: SGNS has been shown to implicitly factorise a shifted PMI matrix.
11. TF-IDF + PMI / PPMI + Cosine Similarity'22'23'24'25 ⚠️ 升溫
三個經典「字-字 / 字-文 關係分數」 — TF-IDF 是「特色詞」分數；PMI 看「兩個字真有緣還是巧合」；Cosine 看「兩個向量方向像不像」。近年趨勢：數值計算題越來越多。

TF-IDF

$\text{tfidf}(t, d) = \text{tf}(t, d) \cdot \log \dfrac{N}{\text{df}(t)}$
$N$ = 總文件數，$\text{df}(t)$ = 含 t 的文件數

直覺：某文常見但別文少見 → 高分（特色詞）；到處都有的字（the, a）→ 低分。

PMI / PPMI

PMI: $\text{PMI}(x, y) = \log \dfrac{P(x, y)}{P(x) P(y)}$ — 同框機率 ÷ 獨立機率乘積
PPMI: $\text{PPMI}(x, y) = \max(\text{PMI}(x, y), 0)$ — 負值不可信，砍為 0

Cosine Similarity — 滿分必背

$\text{sim}(w_1, w_2) = \cos(\vec{w}_1, \vec{w}_2) = \dfrac{\vec{w}_1 \cdot \vec{w}_2}{\|\vec{w}_1\| \|\vec{w}_2\|}$
範圍 $[-1, 1]$；isolates direction from magnitude
滿分守則：寫公式時務必加 sim(w₁, w₂) = ... 在左邊！老師原話：「沒寫 LHS 拿不到滿分。」

  1. Bidirectionality：不像 causal LM（GPT 只看左邊），MLM 強制模型用雙向 context
  2. Pre-train/fine-tune mismatch：[MASK] 只在 pre-training 出現；fine-tuning 沒有。10% 隨機字 + 10% 不變強迫模型對每個位置都計算有用 representation，不只 [MASK] 位置
  3. Prevent trivial copying：10% 隨機替換避免模型只是「複製輸入」
📜 過去考題：'25 Q3(d) — BERT MLM 訓練機制 + masking strategy [4]

滿分答（直接背這段）：

BERT's MLM pre-training objective randomly selects 15% of input tokens for prediction. The model must predict the original token at each selected position.

Masking strategy:

  • 80%: replace with the special [MASK] token
  • 10%: replace with a random vocabulary token
  • 10%: leave the token unchanged

Why this design:

  1. Forces bidirectional context use (vs causal LM which only sees left context).
  2. Mitigates pre-train / fine-tune mismatch: [MASK] never appears during fine-tuning, so the model must learn useful representations at every position.
  3. Random replacement prevents trivial copying.

14. Markov Chain + HMM + Viterbi'20'21'23'25
Markov chain 是「失憶跳棋」— 下一步只看現在。HMM 是「雙層蛋糕」— 上層隱藏狀態（天氣），下層觀察值（冰淇淋銷量），你只看銷量猜天氣。Viterbi 是 DP 偵探，找最可能的隱藏路徑。

Markov Chain（必背 3-tuple）

Markov chain = $(Q, A, \pi)$：
  • $Q$ — set of states
  • $A$ — transition matrix, $a_{ij} = P(q_j \mid q_i)$, rows sum to 1
  • $\pi$ — initial distribution, $\pi_i = P(\text{state}_1 = q_i)$
Markov property: $P(\text{state}_t \mid \text{state}_{1..t-1}) = P(\text{state}_t \mid \text{state}_{t-1})$

HMM（必背 4-tuple）

HMM = $(Q, A, B, \pi)$ — Markov chain over hidden states plus emission matrix $B$：
  • $Q$ 是 hidden（POS tags）
  • Observations are visible（words）
  • $b_i(o) = P(o \mid q_i)$ — emission probability
HMM joint probability: $P(w_1, ..., w_n, t_1, ..., t_n) = \prod_{i=1}^{n} P(w_i \mid t_i) \cdot P(t_i \mid t_{i-1})$
（with $t_0 = \langle s\rangle$）

HMM 為什麼適合 POS tagging？（必背 4 點）

  1. Sequence labelling problem: HMM models 正是 (words, tags) joint distribution
  2. Local dependencies on tags: DT 常後接 NN, MD 後接 VB — first-order Markov 捕捉這個
  3. Words predicted by tag: emission $P(w \mid t)$ 捕捉「the 通常是 DT」這種詞彙偏好
  4. Efficient decoding via Viterbi: $O(N^2 T)$，real-time tagging 可行

Viterbi Algorithm

Viterbi: dynamic programming for the most probable hidden state sequence in $O(N^2 T)$ time.
$v_t(j) = \max_i [v_{t-1}(i) \cdot a_{ij}] \cdot b_j(o_t)$
保存 back-pointers 還原最佳路徑。
"You don't need to be able to use Viterbi step-by-step on the exam. But you need to indicate when it is appropriate."

適用場合：任何 sequence labelling — POS tagging, NER, ASR, gene finding。

📜 過去考題：'21 Q4(b) — Ice-cream HMM Viterbi for [2,2,1] [9]

States: H, C. π=[0.8, 0.2]. P(H|H)=0.7, P(C|H)=0.3, P(H|C)=0.4, P(C|C)=0.6. P(2|H)=0.4, P(2|C)=0.4, P(1|H)=0.2, P(1|C)=0.5.

Label Bias Problem（必背）

Label bias: MEMM normalises probabilities locally per state per observation — states with fewer outgoing transitions are unfairly favoured. In the extreme (one outgoing arc), $P(B \mid A, \text{any obs}) = 1$ — the observation is ignored. Once on a wrong path, the model cannot recover.

CRF 如何解決？

CRF uses global normalisation:
$P(T \mid W) = \dfrac{1}{Z(W)} \exp\!\big(\sum_t \sum_k \lambda_k f_k(t_{t-1}, t_t, W)\big)$
where $Z(W) = \sum_{T'} \exp(\ldots)$ sums over all possible label sequences.

每個 local 決定要跟「全條 path space」競爭 — 出度多寡不再影響 — label bias eliminated.
📜 過去考題：'20 Q4(b) — Label bias + CRF 如何解決 [4]

滿分答（直接背這兩段）：

Label bias problem: MEMM normalises probabilities locally per state per observation — at each step, outgoing probabilities from a given state must sum to 1. This causes states with fewer outgoing transitions to be unfairly preferred: in the extreme case of one outgoing transition, that transition gets probability 1 regardless of the observation, so the observation is effectively ignored. Example: distinguishing "rib" from "rob" — if branched incorrectly at first letter, every subsequent step is locally normalised and cannot redistribute mass to the correct branch.

CRF solution: replaces local normalisation with global normalisation. The partition function $Z(W)$ sums over all possible label sequences, so each local decision is in competition with the entire global path space; observations can re-weight mass across the whole sequence. Label bias is eliminated.

16. CFG + PCFG + Lexicalised PCFG'22'23'24'25
CFG 是「句子製造規則手冊」。PCFG 加上機率 — 多個 parse 時挑機率最高的。Lexicalised PCFG 帶上 head word — 解 PP 接哪裡的歧義。

CFG 定義（4-tuple，必背）

CFG = $(N, \Sigma, R, S)$:
  • $N$ — non-terminals（S, NP, VP, ...）
  • $\Sigma$ — terminals（words）
  • $R$ — production rules of form $A \to \beta$
  • $S \in N$ — start symbol
"Context-free": $A$ 的展開不依賴周圍 context

PCFG

PCFG: CFG with rule probabilities, $\sum_\beta P(\beta \mid A) = 1$.
Tree probability = product of rule probabilities used: $P(\tau) = \prod_{(A \to \beta) \in \tau} P(\beta \mid A)$.

Lexicalised PCFG（解 PP 歧義）

Lexicalised PCFG: each non-terminal annotated with its head word; rule probabilities depend on head words. Resolves attachment ambiguity using lexical evidence.
🌟 經典 PP attachment 例："I shot an elephant in my pajamas"

兩個 valid parses：

  • Parse 1（VP-attach）：I shot (an elephant) (in my pajamas) — 我穿著睡衣射的
  • Parse 2（NP-attach）：I shot (an elephant in my pajamas) — 大象穿著我的睡衣

Plain PCFG 兩者機率相同。Lexicalised PCFG 從語料學到 "shoot ... in pajamas" 比 "elephant in pajamas" 常見，正確選 Parse 1。

📜 過去考題：'24 Q4(d) — Lexicalisation 重要性 [4]

Lexicalisation means annotating each non-terminal with its head word. Rule probabilities are then estimated conditional on these head words.

Importance: plain PCFG rule probabilities are context-free in the lexical sense — they ignore which words actually fill the slots. But linguistic ambiguity is often resolved by specific words: shoot takes "in pajamas" as an instrument far more often than elephant does. Lexicalisation injects this evidence, dramatically improving accuracy on PP-attachment (from ~70% in plain PCFG to ~90%).

17. Chomsky Normal Form (CNF) + CKY Algorithm'25
CNF 是「規矩版 CFG」— 只允許兩種規則形式。CKY 是「由下往上拼樹」的 DP parser，但只接受 CNF。

CNF 定義

Chomsky Normal Form (CNF): 一種限制 CFG，每條 production rule 只能是：
  • $A \to B \, C$（兩個 non-terminals）
  • $A \to w$（一個 terminal）
任何 CFG 都可機械式轉換成 CNF（透過 binarisation 與消除 unit/$\epsilon$ productions）。

CKY Algorithm

CKY: bottom-up dynamic programming parser for CFGs in Chomsky Normal Form. Time complexity $O(|R| \cdot n^3)$ for sentence length $n$.

Core recurrence: $C[i, j] = \{A : A \to BC \in R, \exists k \in [i, j): B \in C[i, k], C \in C[k+1, j]\}$
Base case: $C[i, i] = \{A : A \to w_i \in R\}$

步驟（手算 parse tree 必看）

  1. 填 diagonal：每個 word 對應的 POS 標籤
  2. 填 2-word spans：兩個鄰居能否合併？（rule $A \to BC$ 是否存在）
  3. 填 3-word spans, ... 直到整句
  4. 檢查 $C[1, n]$ 是否包含 $S$
📜 過去考題：'25 Q4(d) — Bottom-up parsing + grammar requirements [4]

CKY (Cocke-Kasami-Younger):

  1. Fill 2D chart $C[i,j]$ indexed by sentence spans.
  2. Diagonal: for each $w_i$, add all $A$ such that $A \to w_i \in R$.
  3. Larger spans: for each split $k$, if $B \in C[i,k]$ and $C \in C[k+1,j]$ and $A \to BC \in R$, add $A$ to $C[i,j]$.
  4. Sentence in language iff $S \in C[1,n]$.

Grammar requirement: must be in Chomsky Normal Form (CNF) — only $A \to BC$ or $A \to w$. Any CFG can be converted. Time: $O(|R| n^3)$.

POS tagging: a sequence-labelling task that assigns each word in a sentence its grammatical category from a predefined tagset (e.g. Penn Treebank, ~45 tags).

Q2 必背公式

Bigram MLE: $P(w_k | w_{k-1}) = c(w_{k-1}, w_k) / c(w_{k-1})$
Add-1 (Laplace): $P = (c + 1) / (c(\cdot) + |V|)$
Add-k: $P = (c + k) / (c(\cdot) + k|V|)$
Good-Turing: $r^* = (r+1) N_{r+1}/N_r$；unseen prob = $N_1/N$
Interpolated KN (必背 3 要素 + 完整公式):
$P_{\text{KN}}(w_i \mid w_{i-1}) = \dfrac{\max(c(w_{i-1}, w_i) - d, 0)}{c(w_{i-1})} + \lambda(w_{i-1}) \cdot P_{\text{cont}}(w_i)$
① discount $d$（typically 0.75）
② $\lambda(w_{i-1}) = (d/c(w_{i-1})) \cdot |\{w: c(w_{i-1}, w) > 0\}|$
③ $P_{\text{cont}}(w) = |\{w': c(w', w) > 0\}| / |\{(w', w''): c(w', w'') > 0\}|$
Perplexity: $PP(W) = P(w_1...w_N)^{-1/N} = 2^{H(W)}$，lower = better

Q3 必背公式

TF-IDF: $\text{tfidf}(t, d) = \text{tf}(t, d) \cdot \log(N / \text{df}(t))$
PMI: $\log [P(x, y) / (P(x) P(y))]$ ; PPMI: $\max(\text{PMI}, 0)$
Cosine（一定要加 LHS）: $\text{sim}(w_1, w_2) = (\vec{w}_1 \cdot \vec{w}_2) / (\|\vec{w}_1\| \cdot \|\vec{w}_2\|)$
P / R / F1:
$P = TP/(TP+FP)$ ; $R = TP/(TP+FN)$ ; $F_1 = 2PR/(P+R)$
Macro-F1: 每類 F1 取 unweighted mean
Micro-F1: 全部 TP/FP/FN 加總再算一次
